# 工作台内容控制与案例库规格（待开发）

状态：W1 / W2 / W3 已实现（2026-07-14）。W2 Migration/RLS 已远端推送。W4 与折叠页仍未开发。

## 1. 目标与不变项

用户需要在左侧精细控制生成类型、长度和语气，并将正反例沉淀为只能由本人选择调用的案例库。

- 保留原文、输入语言、平台、品牌、竞品、结构化写作、创作自由度、粤语程度、中英夹杂、消费者画像、话题日历和参考收藏案例。
- 保留五个平台版本、诊断、审核和消费者反馈。新增文案类型不能把现有五平台结果降级为单一输出。
- 所有新设置必须进入保存配置、云同步、未保存状态判断、生成历史快照和历史载入工作台。
- 旧数据缺失新增字段时必须回退默认值，不能让旧收藏、历史或配置失效。
- 正反例只影响当前用户主动选择的生成请求；不得跨用户读取、注入或展示。

## 2. 左侧上半部分：新增创作参数

### 2.1 文案类型

新增 copyType，默认 social（社媒文案）。

| 值 | UI 名称 | 生成意图 |
|---|---|---|
| social | 社媒文案 | 当前默认能力，适配五个平台 |
| spoken | 口播稿 | 口语化节奏、停顿与可朗读性 |
| poster | 海报短文 | 画面感、短句、强主标题与副标题 |
| advertorial | 软文章 | 信息结构、论点、自然转化 |
| poetry | 诗歌 | 意象、节奏与表达克制 |
| custom | 其他 | 必填 2–20 字类型说明；只作补充，不替代平台与安全约束 |

规则：

1. 新生成的收藏卡片在平台标签左侧、品牌产品字段之后显示文案类型；管理员收藏列表与详情显示相同标签。
2. 旧收藏缺少字段时显示“社媒文案（历史记录）”，不回写既有记录。
3. 每个类型仍生成 standardHK、lightCantonese、IG、Facebook、Shorts，Prompt 按类型和平台共同适配。

### 2.2 可选长度控制

新增 lengthControlEnabled，默认 false；新增 copyLengthLevel，范围 1 至 5，默认 3，仅在开关开启后生效。

- 开关默认关闭，关闭时不向 Prompt 注入长度要求，维持当前输出行为。
- 开启后展示五档滑杆：极短、短、标准、长、加长。
- 服务端按文案类型乘平台给模型软长度目标，不做硬字符截断，以免牺牲粤语自然度、合规说明或 CTA。
- 保存配置、历史、收藏快照均记录开关与数值；旧记录按关闭与标准值恢复。

### 2.3 丰富文案语气

现有语气必须兼容。新模型为一个主语气和最多两个修饰语气。

- 主语气：稳妥、活泼、高级、街坊、年轻、专业、真诚、温暖、幽默、克制、俏皮、激昂。
- 修饰语气：简洁、叙事、促销感、治愈、紧迫、节日感、知识感、对话感。
- 无修饰语气时保持当前单语气效果。已有配置和历史的 tone 映射为主语气，修饰数组为空。

主语气优先；修饰语气只调整节奏、词汇和 CTA 强度，不能覆盖品牌红线、平台规范、粤语程度或案例约束。

## 3. 左侧下半部分：个人案例库

### 3.1 使用方式

新增案例库入口。用户能新建、编辑、删除自己的正例和反例，并在每次生成前选择调用。

| 字段 | 规则 |
|---|---|
| caseType | good（正例）或 bad（反例），必填 |
| title | 选填，最多 120 字；空值不阻止保存 |
| body | 必填，10–3,000 字 |
| reason | 必填，2–500 字，说明为何参考或避免 |
| tags | 可选，最多 8 个，每个 1–20 字 |
| createdAt / updatedAt | 服务端生成 |

交互规则：

1. 当前生成最多选择 3 条案例，正反例可混选；按最近更新时间排序，可按标题、正文、标签和类型检索。
2. 已选择案例显示类型、标题和原因摘要；删除案例前复用现有确认弹窗。
3. 删除后，保存配置中的对应 ID 显示“已删除案例”；加载时忽略并提示，不静默替换。
4. 案例库仅用户本人可读写。普通管理员没有案例正文权限；超级管理员拥有受审计的只读查看与单条复制权限。
5. 标题为空时，用户侧列表显示“未命名正例”或“未命名反例”，并以正文前 24 个字符作为辅助摘要；正文和原因仍为必填。

### 3.2 Prompt 注入规则

- 正例：提取结构、节奏、Hook、用词、CTA 或情绪技巧；禁止复制原文句子、专有事实、品牌名或未经确认的营销承诺。
- 反例：把避免原因转为明确负向约束；禁止复述、模仿或反向拼接反例正文。
- 参考收藏案例与案例库并存，总上下文最多 5 条。其中案例库最多 3 条；超出时优先保留用户刚选择的案例库内容，再按评分保留收藏案例。
- DeepSeek、自部署 CantoneseLLM 和 rules fallback 三条路径必须使用同一结构化案例上下文。未选择案例时输出保持当前行为。

## 4. 数据与同步设计

新增或扩展 AppSettings 字段：

    copyType
    customCopyType
    lengthControlEnabled
    copyLengthLevel
    primaryTone
    toneModifiers
    selectedCaseLibraryIds

- 兼容期保留旧 tone。读取时映射为 primaryTone；新写入仍提供旧生成端可读的兼容语气值。
- saved_configs.config 保存新参数和案例 ID，不保存案例正文副本。
- generation_jobs.brief.workbenchSettings 保存完整参数及已选案例最小快照：ID、类型、标题、正文、原因，以保证案例之后编辑或删除时历史仍可解释。
- 收藏沿用现有 settings 快照写入文案类型、长度和语气。收藏与管理员界面只读快照，不新增跨用户查询。

### 4.1 案例库表与权限

建议新增 case_library_entries：

    id uuid 主键
    owner_id uuid，关联 auth.users.id
    case_type 仅 good 或 bad
    title、body、reason
    tags jsonb，默认空数组
    created_at、updated_at、deleted_at

- RLS：普通用户只能对 owner_id 等于 auth.uid() 的未软删除记录读写。
- BFF：列表、创建、更新、软删除都要求登录，并校验长度、数组和 ID 归属。
- 不新增 public SELECT。普通管理员没有案例正文权限；超级管理员可只读查看案例正文，但每次展开正文或复制必须写 admin_view_case_library_detail 审计日志，审计失败则拒绝正文返回。
- 这张表、迁移与 RLS 推送属于高风险动作，必须在开发前取得单独授权。

## 5. 收藏与管理员界面补充

1. 用户收藏卡片显示顺序：品牌/产品（现有红色）→ 文案类型 → 平台标签 → 时间。
2. 普通管理员收藏列表可查看、展开并单条复制用户已收藏的生成文案正文，同时显示品牌/产品、文案类型、平台、评分、备注、标签、收藏时间。正文返回前必须写审计日志；审计失败则不返回正文。
3. 管理员点击“查看”后，必须在正文上方显示高辨识度的只读审阅摘要卡：品牌、产品、文案类型、平台、评分、用户备注、标签、收藏时间及当前已有的元数据均需突出展示，不能埋在折叠的参数详情中；缺失字段显示“未填写”。摘要卡不提供编辑、删除、评分或导出能力。
4. 管理员收藏详情沿用先审计再读取的现有流程，在正文旁显示文案类型与生成参数摘要；普通管理员保持只读，不能编辑、删除、评分或批量导出。
5. 超级管理员在普通管理员权限之上，额外拥有个人案例库正文的只读查看和单条复制权限；没有案例编辑、删除、批量导出或跨用户数据修改权限。
6. 文案类型、案例库或保存配置不能绕过 Free/Pro 容量、生成额度或 RLS 限制。

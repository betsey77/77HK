# 🎯「灵感参考」功能设计策略

> 核心问题：检索到的 HK 社媒内容，以什么形态呈现？放在哪里？
> 关键约束：用户生成的文案可能是半个月后才发布的，当日热点未必适用——但仍有参考价值。

---

## 一、先厘清：检索到的数据到底有几种？

按「时效衰减曲线」分为四层：

```
高
│  ╱╲
│ ╱  ╲          ← 即时热点（meme/新闻/突发事件/viral post）
│╱    ╲            寿命：几小时到 2-3 天
│      ╲╲╲╲╲╲╲╲╲╲╲╲╲╲╲╲╲╲
│
│  ┌─────────────┐ ← 近期话题（节日/活动/季节）
│  │             │    寿命：2-4 周
│  └─────────────┘
│
│  ┌─────────────────────────────┐ ← 表达模式/语感（怎么写）
│  │                             │   寿命：季度到年度
│  └─────────────────────────────┘
│
│  ┌───────────────────────────────────────┐ ← 竞品情报（别人怎么写）
│  │                                       │   寿命：持续相关
低 └───────────────────────────────────────┘
   现在                                                        未来
   ↑ 即时发布                                                  ↑ 计划发布
```

**核心洞察**：检索回来的数据**不全是「热点」**，分属四类完全不同的东西。每一类都有其独特的参考价值——但不应该被混为一谈。

---

## 二、四大类数据的定位与处理策略

### 🟢 表达模式与语感（永久相关，自动注入）

| 属性 | 说明 |
|------|------|
| 来源 | IG Hashtag 热门帖文、FB 公开主页帖文、YouTube 热门视频 |
| 提取什么 | 句式结构、语气模式、emoji 用法、hashtag 模式、互动引导方式 |
| 寿命 | 季度到年度 |
| Prompt 策略 | **默认自动注入** 2-3 条同行业帖文作为 few-shot 风格参考 |
| UI 展示 | Tab「當下語感」— 展示帖文片段 + 互动数据 |

### 🟡 近期话题（可预测、可规划）

| 属性 | 说明 |
|------|------|
| 来源 | 预制 HK 本地话题日历（节日 + 本地事件 + 行业档期） |
| 提取什么 | 可借势的叙事角度、受众情绪、消费场景 |
| 寿命 | 2-4 周预热期 |
| Prompt 策略 | **基于用户选择的目标发布时间 + 用户勾选**注入 |
| UI 展示 | Tab「話題日曆」— 匹配用户时间窗口的话题列表 |

### 🔴 即时热点（寿命短，但有灵感价值）

| 属性 | 说明 |
|------|------|
| 来源 | IG Hashtag trending、Google Trends HK、YouTube 热门变化 |
| 提取什么 | 此刻正在爆火的表述方式、梗、视觉风格、叙事手法 |
| 寿命 | 1-7 天 |
| Prompt 策略 | **不自动注入**。用户可从面板手动点击「參考表達」复制句式骨架 |
| UI 展示 | Tab「即時熱話」— 带时间衰减标注 |

**为什么即时热点仍然值得做**：

1. **不是所有用户都提前半个月发布**——有些文案是明天就要发的
2. **热点提供「表达灵感」而非「话题抄袭」**——一个 viral post 的写作手法（如特定句式、分段节奏、互动钩子）即使话题过时了，手法仍可复用
3. **品牌借势需要速度**——如果今天突然有个跟品牌相关的事件爆了，用户需要立刻知道并生成借势文案
4. **参考竞品如何追热点**——看竞品此刻在怎么借势，本身就是有价值的竞品情报
5. **用户自己会判断**——系统不需要替用户决定热点是否适用，展示出来，标注时效，让用户自己选

### 🔵 竞品情报（持续相关）

| 属性 | 说明 |
|------|------|
| 来源 | Meta Ad Library（竞品正在投放的广告） |
| 提取什么 | 广告文案、CTA、投放平台、定位话术 |
| 寿命 | 广告投放周期（数周-数月） |
| Prompt 策略 | 注入 audit prompt 作为差异化对比基准 |
| UI 展示 | Tab「競品動態」— 竞品当前活跃广告列表 |

## 三、布局决策：放在 Center Panel 下方，不是 Left Sidebar

### 当前布局问题

```
┌──────────────┬───────────────────┬──────────────┐
│  LEFT (30%)  │   CENTER (flex)   │  RIGHT (30%) │
│              │                   │              │
│ SourceEditor │  5 variant tabs   │  AuditPanel  │
│ LanguageTgl  │  DiagnosisSummary  │              │
│ BrandInput   │  ResultCard ×5    │              │
│ RedLines     │                   │              │
│ Creativity   │                   │              │
│ Platform     │                   │              │
│ Tone         │                   │              │
│ CantoSlider  │                   │              │
│ EnglishSlider│                   │              │
│ PersonaMgr   │                   │              │
│ ConfigMgr    │                   │              │
│ [🚀 生成]    │                   │              │
└──────────────┴───────────────────┴──────────────┘
```

**Left Sidebar 已有 11 个组件**纵向堆叠。再塞入一个 4 Tab × N 条帖文的灵感面板：
- 纵向空间严重不足——用户需要大量滚动
- 横向空间受限——min-width 260px，每条帖文很难展示完整
- 认知负荷过高——输入操作 + 灵感浏览混在一起，注意力分散

### 推荐方案：Center Panel 下方，可折叠

```
┌──────────────┬───────────────────────────┬──────────────┐
│  LEFT (30%)  │       CENTER (flex)       │  RIGHT (30%) │
│              │                           │              │
│ SourceEditor │  ┌─ Tab: IG ─┬─ FB ─┬─... │  AuditPanel  │
│ LanguageTgl  │  │                       │              │
│ BrandInput   │  │   生成文案内容          │              │
│ RedLines     │  │   ...                 │              │
│ Creativity   │  │                       │              │
│ Platform     │  └───────────────────────┘              │
│ Tone         │  DiagnosisSummary                       │
│ CantoSlider  │                           │              │
│ EnglishSlider│  ┌─ 💡 靈感參考 ─────────┐ │              │
│ PersonaMgr   │  │ [當下語感][話題日曆]   │ │              │
│ ConfigMgr    │  │ [即時熱話][競品動態]   │ │              │
│ [🚀 生成]    │  │                       │ │              │
│              │  │  📱 IG #hkfoodie      │ │              │
│              │  │  「是但問個港島人...」   │ │              │
│              │  │  ❤️ 2.3K 💬 186      │ │              │
│              │  │           [複製表達]   │ │              │
│              │  │                       │ │              │
│              │  │  📱 IG #月餅2026       │ │              │
│              │  │  「今年食咗5間...」     │ │              │
│              │  │  ❤️ 1.8K 💬 92       │ │              │
│              │  │           [複製表達]   │ │              │
│              │  └───────────────────────┘ │              │
│              │                           │              │
└──────────────┴───────────────────────────┴──────────────┘
```

### 为什么 Center 下方是对的

| 维度 | Left Sidebar | Center 下方 ✅ |
|------|-------------|---------------|
| 可用空间 | 窄（260px min）→ 帖文展示局促 | 宽（flex-1）→ 帖文可以完整展示 |
| 纵向空间 | 已拥挤，大量滚动 | 结果下方自然延伸，折叠不占位 |
| 认知流 | 输入和灵感混在一起 | 先输入→生成→**看结果→看灵感→迭代**，逻辑连贯 |
| 使用时机 | 生成前（看不到结果） | 生成前后都可以看 |

### 折叠状态

生成前：面板默认折叠（一行标题栏），节省空间。
```
┌─ 💡 靈感參考（展開）─────────────┐
```

生成后：面板自动展开，展示与当前行业/平台相关的参考内容。
```
┌─ 💡 靈感參考（收起）─────────────┐
│ [當下語感] [話題日曆] [即時熱話] [競品動態]
│
│ ...帖文内容...
└──────────────────────────────────┘
```

## 四、四个 Tab 的详细设计

### Tab 1：「當下語感」

> 展示 HK 人当前在社媒上怎么表达——句式、语气、emoji、hashtag，而非话题内容。

**数据来源**：IG Hashtag 热门帖文（P2.8）+ YouTube HK 热门内容（P2.9）+ 参考语料库

**展示方式**：卡片列表，每张卡片 = 一条帖文精华

```
┌──────────────────────────────────────────┐
│ 📱 Instagram · #hkfoodie · 3 日前        │
│                                          │
│ 「是但問個港島人，中秋必買嘅餅舖係？      │
│   留言話我知👇」                          │
│                                          │
│ ❤️ 2,341  💬 186  🔖 #香港美食 #中秋     │
│                                          │
│ [📋 複製表達]  [📝 複製全文]              │
└──────────────────────────────────────────┘
```

**「複製表達」vs「複製全文」**：
- **複製表達**：复制句式骨架（去话题化），如「是但問個___人，___必買嘅___係？留言話我知👇」
- **複製全文**：复制原始帖文全文（含话题）

**排序逻辑**：
- 默认：按「互动率」（engagement / follower ratio）降序
- 可选：按行业相关性优先
- 默认展示 6 条，点击「加载更多」翻页

### Tab 2：「話題日曆」

> 基于用户选择的目标发布日期，匹配前后 2 周内可借势的话题事件。

**数据来源**：预制 HK 本地话题日历（JSON 文件，季度更新）

**展示方式**：时间线 + 话题卡片

## 五、「目標發布時間」字段

支撑 Tab 2「話題日曆」的入口，放在 InputPanel 中：

**位置**：InputPanel 中，BrandRedLinesInput 下方，CreativitySlider 上方

**设计**：
```tsx
<div className="space-y-1.5">
  <label className="text-xs font-medium text-gray-400">
    📅 目標發布時間 <span className="text-gray-600">（可選）</span>
  </label>
  <input
    type="date"
    value={targetDate}
    onChange={(e) => dispatch({ type: 'SET_TARGET_DATE', payload: e.target.value })}
    className="w-full bg-gray-900/60 light:bg-white border border-gray-700/30 light:border-gray-300/50 rounded px-2.5 py-1.5 text-xs text-gray-300 light:text-gray-800"
  />
  {targetDate && matchedEvents.length > 0 && (
    <p className="text-[10px] text-emerald-400 light:text-emerald-600">
      💡 已匹配 {matchedEvents.length} 個可借勢話題（見靈感參考 → 話題日曆）
    </p>
  )}
</div>
```

**行为**：
- 不填 → Tab 2 展示未来 30 天所有话题
- 填了 → Tab 2 仅展示目标日期前后 2 周的话题，其他折叠
- 不影响生成（不自动注入）——用户需要在 Tab 2 中主动勾选想借势的话题

## 六、Prompt 注入策略

不同类型的数据，注入方式不同：

```
Prompt 注入策略
═══════════════════════════════════════════════════════

🟢 表达模式/语感 → 作为 few-shot examples（始终注入）
   触发：P2.8/P2.9 检索 → 语料库匹配 → prompt 注入
   「以下係近期香港 IG 上面高互動帖文嘅表達方式，請參考佢哋嘅
    語氣、句式、互動引導方法——但唔好抄內容：」

🟡 近期话题（用户在 Tab 2 勾选的话题）→ 作为可选叙事角度
   触发：用户选择目标日期 + 勾选话题 → prompt 注入
   「你嘅目標發布時間係 7 月 5 日前後，以下話題適合借勢：
    - 暑假消費旺季 → 可以考慮『暑假限定優惠』角度
    如果適合你嘅品牌，可以融入文案；如果唔適合，忽略。」

🔴 即时热点 → 不自动注入 prompt
   触发：用户手动点击「複製表達」→ 句式骨架进入剪贴板
   用户可自行粘贴到原文输入框，或在手动编辑模式下使用

🔵 竞品广告 → 注入 audit prompt 作为差异化基准
   触发：用户搜索竞品 → audit/re-audit 时自动对比
   「以下係競品目前投放中嘅廣告，請確保你嘅文案：
    1. 唔好同競品過於相似
    2. 找到差異化角度
    3. CTA 強度不低於行業平均」
```

## 八、核心文案策略总结

1. **不要叫「热门文案」**——叫「灵感参考」。不是一个独立的「热门模块」，而是 Center Panel 下方的可折叠参考面板。四个 Tab 让用户按需取用不同类型的参考数据。

2. **即时热点值得做**——但不是推送机制。用户自己判断要不要参考。标注时效衰减，让用户做出知情决策。一个 viral post 的写作手法可以活很久，即使话题本身已经过时。

3. **放在 Center 下方**——Left Sidebar 太挤、太窄。Center 才是「看结果→找灵感→迭代优化」这个认知流的自然栖身之所。

4. **让系统做匹配，不让人做搜索**——Tab 1 自动匹配行业+平台的表达参考。Tab 2 自动匹配目标发布时间的可借势话题。Tab 3 展示此刻正在上升的内容。Tab 4 需要用户主动触发（搜竞品）。

5. **透明但非侵入**——表达模式自动注入 prompt 以提高生成质量（Tab 1），但用户可以在面板中看到「模型参考了什么」。话题（Tab 2）需要用户主动勾选才注入。热点（Tab 3）和竞品（Tab 4）需要用户手动操作。

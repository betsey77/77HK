## 当前配置注意事项

- 本地 health 显示 DeepSeek 已配置，模型为 `deepseek-v4-flash`。
- 本地 `REQUIRE_REAL_MODEL=false` 可用于开发；Preview/Production 必须设为 `true`，避免模型故障时静默以规则结果冒充真实生成。
- `DEEPSEEK_API_KEY` 只能放在 API 项目的 Sensitive 环境变量，不能进入前端、Git、日志或证据。
- 支付仍保持 `PAYMENT_MODE=mock`；本次 2.1 发布不要顺带开启生产支付。

## 推荐发布顺序

1. 用户进行一次统一浏览器产品/视觉验收，其中包含 super_admin 点击 DeepSeek 分析与审阅包反馈。
2. 确认统一验收问题均已关闭。
3. 只读审计 Dirty Worktree，形成明确发布文件清单和分组 commit 方案。
4. 获批后 commit/push release 分支，等待 GitHub CI 全绿。
5. 部署 API/Web Preview，完成真实 Auth、RLS、模型、CORS 和关键业务 smoke。
6. 单独授权生产 Migration；随后 API 先、Web 后 Promote。
7. 生产小流量 smoke 和观察期通过后，再填写 v2.1 更新日志。
